# TalentFlow — Features Reference

This document describes the features, components, and developer workflows for the TalentFlow job board.

## Overview
TalentFlow is a minimal, modern job board for AI roles built with Next.js and Tailwind-style utilities. The UI emphasizes clarity, accessibility, and fast scanning.

## Pages
- Home (`/`)
  - Hero with search, popular tags, and quick site stats.
  - Featured roles preview and category pills.
- Jobs listing (`/jobs`)
  - Search and filters (query + location).
  - Paginated or grid list of job cards.
  - Breadcrumbs and page title reflecting filters.
- Job detail (`/jobs/[id]`)
  - Full job description, metadata, and badges.
- Post job (`/jobs/new`)
  - Simple job submission form (title, company, description, location, salary, tags).

## Key Components
- `SiteHeader` / `SiteFooter` — global navigation and branding. Header is sticky with a subtle blur and semi-transparent background.
- `HeroSection` / `HeroSearch` — prominent hero with search input and CTA buttons.
- `JobFilters` — compact filter panel with search input, location selector, and `Search` action.
- `JobList` / `JobCard` — responsive grid of cards. Each card includes title, company, description (clamped to 3 lines), badges, location, salary, and posted date.
- `JobBadge` — pill-style labels used for categories, job type, and focus area (accent variant available).
- `JobsEmptyState` — friendly empty and no-results states with actions to clear filters, browse or subscribe to alerts.
- `SectionHeading`, `Button`, `Input`, `Select`, `Textarea` — shared UI primitives for consistent spacing and typography.

## Visual & Accessibility Notes
- Color tokens and utilities are defined in `app/globals.css` using CSS custom properties (`--accent`, `--muted`, `--foreground`, etc.).
- Card surfaces use large, soft shadows and slight lift on hover for a premium feel.
- Job titles now wrap to a second line; descriptions are clamped to 3 lines for consistent card heights.
- Placeholder text and muted copy have increased contrast to improve readability (WCAG-conscious adjustments).
- Buttons follow pill or rounded styles and use subtle glow on primary accent.

## Behavior & UX
- The header remains sticky on scroll for quick access to navigation and 'Post a job'.
- Searching uses the jobs filter form and updates the listing page via query params; clearing filters returns to a broad listing.
- Empty states include suggested actions and a `mailto:` subscribe fallback for job alerts.

## Developer Notes
- Layout: `app/layout.tsx` sets fonts and global styles (uses Geist fonts via `next/font`).
- Container: `components/layout/container.tsx` centralizes max-width and horizontal padding.
- Styling: Tailwind utility classes are used alongside a local `@layer utilities` block in `app/globals.css` for project-specific tokens.
- Linting: `npm run lint` runs ESLint (config present in repo).

## CI / Deployment
- GitHub Actions workflow: `.github/workflows/ci.yml` runs lint and `npm run build`. On `main`, it uploads artifacts and deploys to Vercel via `vercel/action@v3`.
- Required repository secrets for deployment:
  - `VERCEL_TOKEN` — Vercel personal token
  - `VERCEL_ORG_ID` — Vercel organization or team ID
  - `VERCEL_PROJECT_ID` — Vercel project ID

## Local Development
1. Install dependencies:

```bash
npm install
```

2. Run dev server:

```bash
npm run dev
# opens at http://localhost:3000
```

3. Lint and build locally:

```bash
npm run lint
npm run build
```

## Extending the App
- Add new badges, filters, or fields in `components/jobs/*` and update `lib/jobs.ts` to persist or index new properties.
- To change theme tokens, edit `:root` variables in `app/globals.css`.
- For internationalization, wrap job text and dates with an i18n library and provide translations for UI strings.
